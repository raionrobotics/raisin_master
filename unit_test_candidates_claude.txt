================================================================================
주요 소스:

---------- 단위 테스트 후보 ----------

[6-U-1] Lie 군 연산 (LieGroup.cpp)
  - Exp_SO3: 영 벡터(‖w‖ < TOLERANCE=1e-10) → 단위행렬
  - Exp_SO3: 알려진 회전 벡터 → 회전행렬 수학 검증
  - Exp_SEK3: K=2일 때 블록 구조 정확도
  - skew(): 반대칭 행렬 성질 (A = -A^T) 확인

[6-U-2] 상태 전파 (IEKF::Propagate — SE_K(3) 스트랩다운 적분)  [수정]
  - R_pred = R * Exp_SO3(ω*dt) 수학 검증
  - v_pred = v + (R*a + g)*dt 검증 (중력 [0,0,-9.8] 포함)
  - p_pred = p + v*dt + 0.5*(R*a+g)*dt² 검증
  - 바이어스 보상 전/후 적분 차이 확인
  - 공분산 전파 후 P 양정치(PD) 유지 확인

[6-U-3] 측정 갱신 (IEKF::Correct)  [신규: 원본 누락]
  - 갱신 후 P 공분산 감소 확인

---------- 단위 테스트 후보 ----------

[13-U-1] 프레임 트리 탐색 (canTransform)
  - 단순 체인 A→B→C: canTransform(A, C) = true
  - 공통 루트: A→X←B: canTransform(A, B) = true
  - 분리된 프레임: canTransform(A, D) = false (연결 없음)
  - 동일 프레임: canTransform(A, A) = true
  - 사이클 감지 (frame == parent 조건): A→B→A → false 반환 + 무한 루프 없음

[13-U-2] 변환 합성 (getTransform)
  - 체인 5개 프레임: 최종 변환 수학 검증
  - 단위행렬 합성: T * I = T
  - 역변환: getTransform(B, A) ≈ getTransform(A, B)^{-1}
  - 분기 병합: 공통 루트 경유 합성
  - 루트가 다른 두 프레임: Identity 반환 + 경고 로그 발생  [신규]

[13-U-3] 정적 변환 체인 처리 (setTransform + is_static=true)
  - 정적 체인 A_s→B_s→C에 대해 동적 TF 설정 → static root(A_s)에 저장 확인
  - T_child_to_root 보정 인수 적용 수학 검증

[13-U-4] geometry_msgs 변환 (toIsometry)
  - 단위 사원수 → 단위행렬
  - 90° 회전 사원수 → 회전행렬 수학 검증
  - 비정규화 사원수 처리  [수정: 실제 코드에 정규화 없음 → 동작 정의 필요]
    비고: 현재 구현은 Quaternion(w,x,y,z).toRotationMatrix() 직접 호출로
    정규화 검사 없음. "비정규화 입력 시 예상 동작"을 먼저 스펙으로
    정의해야 테스트 작성 가능.

[13-U-5] 다중 로봇 프레임 공존
  - 두 로봇의 프레임이 별도 닉네임으로 분리되어 공존
  - 각 로봇의 프레임 간 변환은 공통 월드 루트를 통해 가능

[13-U-6] setTransform 덮어쓰기 동작  [신규]
